# docs/product/

End-user and developer **documentation of the built application**, produced by the Docs role once
features are released. One subfolder per feature: `docs/product/<slug>/`, authored from
`docs/templates/documentation-template.md` and published to the Confluence Documentation folder.

This is distinct from the rest of `docs/`, which documents the AI *environment* itself (roles,
process, standards, templates).

_(empty until the first feature ships)_
