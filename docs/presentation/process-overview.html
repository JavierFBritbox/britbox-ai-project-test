The idea

A team of roles, not a single bot

Each role is a Claude agent with a narrow job and its own tools. They hand off through versioned artifacts in the repo — and pause at explicit human gates.

Where humans decide

Four sign-off gates

Everything else runs automatically. Humans control what to build and the solution shape; AI owns turning it into shipped code.

  • Product sign-off — a human sets the requirement's Confluence status to Signed Off.
  • Architect agreement — the human architect signs off the AWS design.
  • Tech Lead sign-off — the human approves the code design and the ticket breakdown.
  • DevOps deploy-rule approval — the human agrees environments, target AWS account, and deploy rules.
Plus out-of-band human work: authoring requirements, driving the interactive interviews, and setting up cloud IAM / OIDC (AI never escalates permissions).

Guardrails

Non-negotiable rules

  • Config-driven integrations. Every Atlassian ID comes from config/atlassian.json; skills refuse live calls while unconfigured.
  • Jira gate. No role mutates anything without a ticket moved To Do → In Progress → In Review → Done.
  • Per-feature artifacts. One folder per feature; never overwrite another feature's files.
  • Author / review separation. The authoring pass and the verification pass stay distinct — verify before done.
  • Enforced DevOps standards. GitHub Actions + AWS/Terraform only; OIDC, no long-lived keys; humans own cloud permissions.
